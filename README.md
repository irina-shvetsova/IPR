# ИПР·AI — MVP генерации индивидуального плана развития

Streamlit-прототип, который собирает ИПР по результатам опроса 360° и ответам
сотрудника. Повторяет принятый пайплайн проекта и стиль дельтовского референса.

Сквозной сценарий:

1. **Загрузка 360°** — CSV-выгрузка разбирается в структурированный профиль.
2. **Экраны выбора** — направление движения (вертикально / горизонтально),
   три ожидания от работы через год, готовность вложиться.
3. **Генерация** — один структурированный промпт к DeepSeek → JSON → рендер.
4. **Выгрузка** — DOCX по фирменному формату и ICS-календарь точек контроля.

В промпт зашиты все редакторские правила, включая правки последнего цикла:
вводный текст в начале, контекст развития из ответов сотрудника, риски внутри
каждого направления (отдельного раздела рисков нет) с поддерживающей рамкой,
указанный источник обучения (дельтовская библиотека), сроки по циклам, а не по
датам, и точки контроля с нарастающей интенсивностью и рефлексивными вопросами.

## Локальный запуск

```bash
pip install -r requirements.txt
cp .streamlit/secrets.toml.example .streamlit/secrets.toml   # заполните доступ к Яндекс Клауду
streamlit run app.py
```

DeepSeek подключён через Яндекс Клауд (AI Studio). Реквизиты можно не класть в
secrets, а ввести вручную в боковой панели: API-ключ сервисного аккаунта,
folder_id и id модели. Загрузка 360° и экраны выбора работают без них — доступ
нужен только для генерации.

## Развёртывание на Streamlit Cloud

1. Запушить репозиторий на GitHub (ветка `main`).
2. На share.streamlit.io → **New app** → выбрать репозиторий, файл `app.py`.
3. **App settings → Secrets** → вставить:
   ```toml
   YANDEX_API_KEY  = "AQVN..."
   YANDEX_FOLDER_ID = "b1g..."
   YANDEX_MODEL_ID = "deepseek-v3"
   ```
4. Deploy. Обновления подтягиваются автоматически при push в `main`.

Для внутреннего контура (корпоративная сеть) разверните тот же код на своём
хостинге; реквизиты держите в переменных окружения или в `.streamlit/secrets.toml`.

## Структура

```
app.py                — Streamlit-интерфейс: загрузка, экраны выбора, генерация
preprocessing.py      — разбор CSV 360° в структурированный профиль
ipr_generator.py      — клиент DeepSeek, промпт с правилами, структурированный вывод
renderers.py          — рендер DOCX (python-docx) и ICS (icalendar)
requirements.txt      — зависимости
sample_360.csv        — пример входного файла для проверки
.streamlit/           — тема оформления и пример secrets
```

## Формат входного CSV

```
тип,название,оценка,комментарий
компетенция,Профессионализм,9.5,
деструктор,Бескомпромиссность,4.2,
роль,Советник,10.0,
```

Колонка `тип` отличает компетенции от деструкторов и ролей. Числа принимаются
и с точкой, и с запятой. Кодировки utf-8 и cp1251, разделители «,» «;» «таб».

## Задел под RAG

Профиль и ответы сотрудника собираются в структурированные объекты, что
позволяет позже подключить хранилище (pgvector / Qdrant) и поиск похожих
случаев без переработки пайплайна. На текущем этапе RAG не вводится — все
данные помещаются в контекст одного запроса.

## Заметки по подключению DeepSeek через Яндекс Клауд

DeepSeek подключён через Yandex AI Studio (Foundation Models). API OpenAI-
совместимый, поэтому клиент — `openai.OpenAI` с `base_url="https://llm.api.cloud.yandex.net/v1"`,
ключом сервисного аккаунта и `project=<folder_id>`. Модель адресуется через URI
`gpt://<folder_id>/<model_id>/latest`. Точный `model_id` для DeepSeek берётся из
Model Gallery в консоли Яндекс Клауда (по умолчанию в коде `deepseek-v3` —
проверьте актуальное имя). Температура 0,4 — компромисс между связностью и
стабильностью стиля; если стиль «плывёт», снизьте до 0,3.

Что нужно настроить в Яндекс Клауде (сервер разворачивать не требуется):
каталог (folder), сервисный аккаунт с ролью `ai.languageModels.user`, API-ключ
для него, и доступ к нужной модели в Model Gallery. Логирование запросов
отключается заголовком `x-data-logging-enabled: false` — он уже проставлен в коде.
